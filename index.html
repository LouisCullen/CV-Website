Honest Manchester

Honest Manchester was a site created for university students in Manchester to submit anonymous confessions about their experiences. Intended to give a more anonymous alternative to social media confession pages, the idea was to require a confession from the user to enter the site and view others. This would ensure a consistent flow of content.
This project was unique for me as it was undertaken as a method of learning new technologies which I had not experienced before. Because of this, the whole project took an experimental approach and was not intended to be particularly polished or display best practices. Instead, the goal was to have a working site which would teach me some techniques as it was created.
The site was built on a stack of MongoDB, ExpressJS, React and NodeJS, extending my JavaScript abilities beyond what I had experienced before. The familiar frontend styling gave me an opportunity to implement better practices such as less inline styles and less reliance on Bootstrap. Meanwhile, my new exposure to a full stack got me more comfortable working with backend logic.

hustl.io

Hustl.io was a team project completed in my first year of university. Throughout the year we visualised, designed and produced a fantasy trading website with real stock market data and inter-user functionality. My main tasks were front-end development and implementation of the inter-user friend system.
This project was beneficial for me as it was my first experience of working as a team on a programming project. It taught me how important communication was, with our use of discord calls and regular meetings keeping the project on track, as well as the importance of being open about lack of knowledge and learning from other developers.
The technologies used were Python, Django, CSS and HTML. I also encountered Docker, SQLite and Postgres DB during development, though these were more the focus of other members of the team.

Asteroid Belt

Asteroid Belt was a solo project focused on game development. On top of the retro styling being difficult to get right, the challenges surrounding implementation stretched my logical ability and forced me to find workarounds where there was no native solution.
Creating the game was enjoyable as it gave me a creative freedom which I hadn't experienced much before. The prompt gave some requirements for game features, but the vagueness of the rest allowed me to manage my own time and create a product fully from my own vision.
The single programming language used was Python with its Tkinter library, which was a given requirement. Tkinter isn't designed for game development like the Pygame library is, so it forced me to find workarounds for implementation, such as my own collision detection and entity tracking.

lc-computing.com

lc-computing.com is this project, designed as a more creative approach to a CV while showing some of my web development abilities too. Creating this website has challenged me to discover new techniques to make a professional-looking website, while still giving me a platform to show my preferred styling and personality and show off new techniques I pick up.
The main difference between this project and those I've done before is the lack of external motivation. As it is not being created for anyone else, all motivation must come from myself and any delays only impact my online profile. It has also challenged me to choose when to roll out new versions in order to avoid a messy site going live.
Created with CSS, HTML, JavaScript and Bootstrap, the website hosted as a static website on AWS through their S3 storage system. With the DNS also handled by AWS' Route 53, I have had to research different methods of hosting websites with cloud computing to find what worked for me, giving me further knowledge outside any courses I have done.
